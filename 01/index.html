Avoid signups from used addresses

The user’s transaction history across dapps shouldn’t be public.

You’re responsible for safeguarding the user’s privacy. Assume the data on the blockchain will remain there forever, and that it will be trivial to correlate user-operated addresses through their transaction history. Avoid linking addresses with a history to anything that contains the user’s personal information.

Early in your dapp’s onboarding process, check whether the public address that the user is signing up with has been used before.

Case 1: your signup is free or subsidized

If your signup uses an off-chain signature validation or you’re going to cover the costs of storing the user’s data on the blockchain, you should ask the user to switch to a new address. After she does, you can complete the signup.

Signing up for CryptoKitties is free and done off-chain. They ask the user's web3 provider to sign a message to prove it has the private key to his publicly-exposed Ethereum address.


In this case, CryptoKitties should check if the address 0xfdff... has been used for other individual transactions. If so, it should require the user to sign up using a different one, giving user-specific examples of the data correlations that would be exposed.


Case 2: the user needs to pay for her signup

If the user needs an address with funds to store some signup data on the blockchain, your safest bet is to ask her to switch to a new address, fund it through an exchange or another contract, and finally come back to your dapp to complete the signup. This introduces a lot of friction, but is especially required in dapps that store personal information.

Signing up for Bloom requires the user to complete a blockchain transaction from his web3 provider. Sensitive data is stored in this step, so the right thing to do is to demand a fresh address that's funded by an exchange.


Remember: general warnings aren't enough

Even if you warn that Ethereum addresses and transaction histories are public, the user will rarely stop to think about where he's used his account before. At the very least, your dapp should expose what it knows about the address being used so the user makes an informed decision about whether to use it.
